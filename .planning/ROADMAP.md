# Roadmap: Thai Election 2026 Data Pipeline (Uthai Thani 2)

## Overview

This roadmap defines the sequential progression from setting up the initial OCR prototyping environment to fully extracting 250+ polling stations' PDF results, culminating in a comprehensive data science analysis and dashboard addressing local political insights.

## Phases

- [x] **Phase 1: Prototyping Data Prep Pipeline** - Setup notebook environment and prototype OCR extraction on sample PDFs. (completed 2026-04-09)
- [ ] **Phase 2: Scaled Extraction & Validation** - Automate processing across all stations, validate data consistency, and export CSV.
- [ ] **Phase 3: Exploratory Analysis & Dashboard** - Generate insights (geo-anomalies, political dynasty shifts, past references) via analytical visuals.

## Phase Details

### Phase 1: Prototyping Data Prep Pipeline
**Goal**: Verify the effectiveness of the selected Thai OCR engine (Typhoon OCR vs PaddleOCR) inside an `ipynb` environment using a tiny sample before scaling up.
**Depends on**: Nothing (first phase)
**Requirements**: EXTR-01, EXTR-02, EXTR-03, EXTR-04
**Success Criteria**:
  1. Notebook successfully loops through a small folder subset.
  2. OCR engine correctly recognizes text blocks without garbling Thai characters.
**Plans**: 1 plan

Plans:
- [x] 01-01: Prototyping OCR extraction and pdf2image loading.

### Phase 2: Scaled Extraction & Validation
**Goal**: Execute the OCR across all PDFs for Uthai Thani 2, clean common anomalies, and validate against logical vote sum totals. Export structured dataset.
**Depends on**: Phase 1
**Requirements**: EXTR-05, DATA-01, DATA-02, DATA-03
**Success Criteria**:
  1. A structured dataset (`election_structured_data.csv`) is generated containing valid vote totals.
  2. Clean and validated data rules assert total votes successfully.
**Plans**: 2 plans

Plans:
- [x] 02-01: Scale extraction mechanism and robust bounding box logic.
- [ ] 02-02: Implement cleaning rules and save export file.

### Phase 3: Exploratory Analysis & Dashboard
**Goal**: Utilize the generated dataset to answer the deep socio-political hypotheses generated by the user (dynasty tracking, ratios, anomalies, PAO interference).
**Depends on**: Phase 2
**Requirements**: ANLY-01, ANLY-02, ANLY-03, ANLY-04, ANLY-05, ANLY-06, ANLY-07, ANLY-08, ANLY-09, ANLY-10, ANLY-11, DASH-01
**Success Criteria**:
  1. All 11 hypothesis markers are explored successfully with visual aids.
  2. The notebook effectively functions as an analytical dashboard.
**Plans**: 3 plans

Plans:
- [ ] 03-01: Foundational EDA, correlations (PAO, ballot numbers) and baseline charts.
- [ ] 03-02: Advanced analysis (vote buying outliers, vote banking ratios, party sizes).
- [ ] 03-03: External integration contexts (dynasty trends, geospatial mapping, Google/News trends).
